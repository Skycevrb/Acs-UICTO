Who We Are

The Association of Cybersecurity Students (ACS) at UICTO is a student-led organisation dedicated to building practical cybersecurity skills, community, and professional readiness for students of the Ebonyi State University of ICT, Oferekpe, Nigeria.

From our very first lesson — setting up Git Bash and running git init — we treat every project as a professional software development exercise. That means proper commit history, meaningful commit messages, secure secret management, and never pushing without running git diff --staged.

Under the guidance of Mr Nwebonyi Harrison, our curriculum covers Version Control Systems (CYB-100), penetration testing, network security, IAM best practices, and ethical hacking — grounding theory in hands-on labs.

STAGE

git add

Moves changes from your working directory into the staging area — the inspection table before a commit. Always use git add -p to review each change individually. Never use git add . blindly.

$ git status
modified: index.html
modified: .env ← DO NOT STAGE!
 
$ git add -p index.html
Stage this hunk? [y/n]
 
$ git diff --staged
✓ only index.html staged
COMMIT

git commit

Permanently saves the staged snapshot into the local repository with a cryptographic SHA-1 hash. Write commits using the Conventional Commits format so your history is human-readable and machine-searchable.

# ✓ Good — IAM-searchable message
$ git commit -m "feat(nav): add mobile hamburger menu"
[main b7d4c82] feat(nav): add mobile...
 
# ✗ Bad — defeats the audit trail
$ git commit -m "update"
← analyst cannot find this in log
PUSH

git push

Uploads your local commits to a remote repository (e.g., GitHub), making your work available to the rest of the team. Before pushing to main, ensure all tests pass and pre-commit hooks have cleared.

# First push — set upstream branch
$ git push -u origin main
Branch 'main' set to track 'origin/main'
 
# Subsequent pushes
$ git push
To github.com:ACS-UICTO/website.git
b7d4c82..f9a3c11 main → main
IGNORE

.gitignore

A plain text file that tells Git which files to never track. Create it before your first commit. Protect .env, *.key, *.pem, and *.log at minimum.

# .gitignore — create FIRST
.env ← secrets protected
*.key *.pem ← keys blocked
*.log ← logs excluded
node_modules/ ← dependencies skipped
 
$ git status
.env — not tracked ✓
BRANCH

git branch

Branches let you work on a new feature or fix in isolation — without breaking the main branch. Think of branches as parallel universes of your codebase that can later be merged together.

# Create and switch to a new branch
$ git checkout -b feat/mobile-nav
Switched to a new branch 'feat/mobile-nav'
 
$ git branch
* feat/mobile-nav ← current
main
 
$ git merge feat/mobile-nav
AUDIT

git log

The forensic tool of Git — shows the complete commit history (your IAM audit trail). Use flags to filter by author, date, file, or message keyword. Essential for security investigations.

$ git log --oneline --graph --all
* f9a3c11 feat(nav): add hamburger menu
* b7d4c82 security: install pre-commit hooks
* a3f9c12 chore: add IAM-safe .gitignore
 
# Search by keyword
$ git log --grep='security' --oneline

# Find all security-related commits
$ git log --grep='security' --oneline
 
# Find who last changed the login page
$ git blame auth/login.js

$ git log -S 'sk_live_' --oneline
